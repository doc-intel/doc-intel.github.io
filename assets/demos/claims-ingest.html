Claims ETL

From a messy client Excel to a clean base: detection, column recognition, validation, normalisation.

Static demo: the path below is frozen on the demo file claims_sample.xlsx (19 claims, 7 columns). No server. For the interactive tool (upload, confirmation, export), run the Streamlit app in ../app/.

1 The file arrives… messy

Headers with trailing spaces, a line break, inconsistent labels, one corrupted date column.

Header receivedProblem
'Notification date 'trailing space
'Loss\n Date'line break inside the header
'Payment Amount 'trailing space + floats to round
'Covid claims 'values covid vs cvid (typo)
'Loan Type'3 spellings: Car loan / Car Loan / Car Loan
'Effective date'every value = 1905-07-11 (corrupted Excel serial)

2 Column recognition

Each header is cleaned then matched to the target schema by similarity. A confidence score, to be confirmed by the client.

Source columnTarget fieldConfidence
Notification date→notification_date🟢 sure
Loss Date→loss_date🟢 sure
Payment Amount→payment_amount🟢 sure
Loss Type→loss_type🟢 sure
Covid claims→covid_flag🟢 sure
Loan Type→loan_type🟢 sure
Effective date→effective_date🟢 sure

3 Amount & date validation

Amounts are read whatever the separators (comma/dot, symbols). Out-of-range dates are flagged, not imported silently.

FieldVerdict
payment_amount19/19 recognised
notification_date, loss_date19/19 recognised
effective_date⚠️ 19/19 = 1905-07-11 — suspicious dates (Excel serial), to check before import

4 Label normalisation

We group the spellings of a same label and propose a canonical one. The client keeps control: they know their labels best.

FieldSpellings receivedProposed canonical
loan_typeCar loan, Car Loan , Car Loan→Car Loan case/spacing
covid_flagcovid (18), cvid (1)→covid close spelling
loss_typeDeath Any Cause (17), DAC (1)→Death Any Cause acronym

Typed columns, numeric amounts, unified labels, suspicious dates flagged. Ready for Excel/CSV export and for the SQL agent.
