PII redaction on a synthetic CV (regex layer)

Detects 2 PII entities on a 26-line resume via the pipeline's regex cascade.

Synthetic CV (Sarah Mitchell, fictional identity). The regex layer catches structural PII (email, phone, IBAN, SSN, credit card). Names / addresses would need the LLM layer of the cascade.

INPUT — what we fed the pipeline

Document
cv_classic.pdf
Ground truth
sarah.mitchell@example.com, +44 7700 900123 (from cv_clean.gt.md)
Line count
26 lines

PROCESSING — what the pipeline does

  1. parse_pdf(cv_pdf) → line_df
  2. detect_regex(line_df) → list[RedactionMark] via 5 patterns (IBAN, SSN, CC, email, phone)
  3. merge_marks(marks) → dedupe overlapping matches by category priority

OUTPUT — what came back

2
Categories
pii_email, pii_phone
pii_email       L3: 'sarah.mitchell@example.com'
pii_phone       L3: '+44 7700 900123'
