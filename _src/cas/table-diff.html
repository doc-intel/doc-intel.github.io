Diffing two versions of a portfolio table

Two exports of the same portfolio, a month apart. A raw file diff is useless on a table. What you want is per-row: which positions appeared, which vanished, which values moved, and by how much.

The problem

Tabular data changes between versions: rows added, rows dropped, values updated. A text diff on the exported file is noise (reordering, formatting) and misses the point. The question is a business one: what changed in the portfolio, position by position.

The approach

The two tables are aligned on their key, then compared cell by cell. The output is a structured set of changes: added rows, removed rows, and modified cells with their old and new values. Reordering and cosmetic differences do not register; only real changes do.

The thread: a table diff is aligned on the key, not on the text. The result is a list of real, per-row changes a reader can act on.

What it shows

Comparing tables is a different operation from comparing text. Aligning on the key turns a noisy file diff into a clean, per-position change list.

Deliverables

A change report between two versions of a portfolio table: added, removed and modified rows with their values. The portfolios are synthetic.
