Ingesting heterogeneous claims files

Claims arrive as spreadsheets that never quite match: different headers, mixed date formats, labels that drift. Before any analysis, they become one clean typed table.

The problem

Every source exports claims differently. Headers sit on the wrong row, amounts mix separators, dates come in three formats, the same label is spelled five ways.

The approach

The pipeline detects the real header row, maps each column to a target field, hardens amounts and dates into typed values, and normalizes labels against a reference. The output is one clean typed table, with the mapping kept so any step is auditable.

The thread: ingestion is where trust is won or lost. A typed table with a recorded mapping beats a clever model on dirty inputs.

What it shows

Most of the value in a claims pipeline is upstream of any model: a heterogeneous pile of files becomes one queryable table, deterministically.
