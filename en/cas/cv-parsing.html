Turning CVs into Markdown: four methods compared

Docling, Azure Document Intelligence, GPT-4.1 and GPT-5-nano on synthetic CV images, scored against ground truth, with the failure modes mapped per layout.

The problem

"Just OCR the CV" hides a hard question: which method, on which layout? A single-column CV, a two-column CV with a sidebar, a dense academic format, a Europass table, a degraded scan, each breaks a different tool in a different way. Without a controlled test, you pick a parser on vibes.

The approach

Five synthetic CVs are generated to cover five layout archetypes, each with its own ground-truth Markdown. Four methods run on all five: local OCR (Docling), cloud OCR (Azure Document Intelligence), and two vision-language models (GPT-4.1 and GPT-5-nano). Every output is scored against ground truth with a simple, transparent word-level similarity, and the failures are read qualitatively per layout.

Everything here is synthetic: the CVs are generated, so there is no personal data. That is also what makes the benchmark repeatable, the ground truth is known exactly.

The method, step by step

Generate synthetic CVsFive layout archetypes built programmatically, each with a ground-truth Markdown.
Run four methodsDocling locally, Azure DI, GPT-4.1 and GPT-5-nano on each CV.
Score against ground truthWord-level similarity after lowercasing and stripping punctuation.
Read the failures per layoutVision models tend to destroy tables; OCR tends to scramble sidebars.
Publish the evidenceRaw outputs, a side-by-side verification view, and a scoring matrix.

What it shows

There is no single best parser, only a best parser for a given layout. A vision model reads a degraded scan a local OCR cannot, but flattens a clean table a layout parser would preserve. The lesson feeds straight into the parsing brick of the series: escalate deliberately, and match the method to what the document actually is.

Artifacts

Twenty method-by-CV outputs, per-CV scoring reports, the synthetic images with their ground truth, and a written analysis of the failure modes.
