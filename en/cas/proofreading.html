Proofreading a document, error by error

A model can rewrite a paragraph and quietly change its meaning. A safer proofreader returns a typed list of corrections, each pinned to where it applies, so a human keeps control of what actually changes.

The problem

Ask a model to "fix" a document and it hands back a rewritten version. You then have to diff the two to see what changed, and hope it did not alter a number, a name or a clause while it was at it. In a regulated setting that is not acceptable: you need to know exactly what was flagged and why.

The approach

The proofreader reads the document and returns a structured list of issues: the error, the suggested correction, and where it sits. Nothing is rewritten in place. A reviewer reads the list, accepts or rejects each correction, and stays in control of the final text.

The thread: proofreading returns a list of typed corrections, not a rewritten blob. The human decides what changes; the tool only points.

What it shows

Correction is safer as a list than as a rewrite. Each issue is auditable, and the document only changes where a human agrees it should.
