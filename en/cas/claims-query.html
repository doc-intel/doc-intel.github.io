Querying a claims database in plain English

How many water-damage claims over 5,000 this year? The agent turns the question into a bounded SQL query, runs it, and shows the query as much as the result.

The problem

The handler knows the business question but not the schema or SQL. The analyst rewrites the same queries all day. Between them, someone has to turn plain English into a correct, safe query.

The approach

The agent knows the claims table schema and generates a read-only SELECT. The query is bounded (no writes, validated tables and columns), executed, and returned with the generated SQL in plain sight.

The thread: the agent only translates intent into bounded SQL. The query is shown, so it is verifiable, not a black box.

What it shows

A table does not need fuzzy semantic search to be queried: once typed, the question drives a deterministic query, far more reliable than approximate text search.

Deliverables

An interface that takes an English question, shows the generated SQL and the result. The claims data is synthetic.
