Catching a tampered invoice before it is paid

The same repair invoice, altered three ways. The fraud is not in what the invoice says, it is in how the file was made: a desktop editor, a modification weeks later, and an earlier version buried inside the PDF.

The problem

A claims handler receives an invoice as a PDF. It looks fine. But a number may have been changed after the fact, a total inflated, a document re-saved to hide an edit. Reading the printed page tells you nothing about this: the proof of tampering is in the file, not on it.

The approach

The detector reads the document itself, not just its text. It looks at the metadata (which software produced it, when it was created versus modified, timezone mismatches), and at the PDF's revision history (an incremental save can leave an earlier, honest version buried inside). Each signal is circumstantial on its own, so the verdict is graded: green when nothing fires, amber when the evidence is only circumstantial and deserves a human review, red when a document was demonstrably edited after creation with the earlier state still inside it.

The thread: one metadata flag is a reason to look, not an accusation. A red verdict needs a document that was edited after creation with the pre-edit state still recoverable — two independent facts pointing the same way.

The method, step by step

Read the metadataProducer software, creation vs modification dates, timezone mismatches — the first thread to pull.
Inspect the revisionsAn incremental save can leave an earlier version buried in the PDF; recovering it is direct proof of an edit.
Grade the verdictGreen (clean), amber (circumstantial, review), red (edited after creation with the earlier state inside).
Keep it deterministicMetadata and revisions are read from the file, no model needed; the answer is reproducible.

What it shows

Fraud detection on documents starts before any language model: the file's own structure carries the evidence. A pure arithmetic fraud (a total padded above the line items) is caught by a separate detector that first extracts the line items — a different path to the same kind of red flag.
