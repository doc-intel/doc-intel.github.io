484 supports d'investissement, extraits puis interrogeables

Extraire les supports d'un contrat d'épargne, les enrichir, puis laisser un agent traduire une question en français vers une requête SQL bornée.

Le problème

Les conditions générales d'un contrat d'épargne listent des centaines de supports d'investissement, noyés dans des dizaines de pages. Un conseiller qui cherche « les fonds actions Europe ISR avec des frais faibles » n'a aucun moyen de filtrer cette liste : le PDF n'est pas une base de données.

La démarche

On extrait les 484 supports du PDF vers une table typée (CSV, JSON, Excel, SQLite). On enrichit ensuite chaque support avec un LLM qui lit son intitulé et son objectif pour en déduire la classe d'actifs, la zone géographique, le caractère ISR, l'éligibilité PEA et des mots-clés. Enfin, un agent reçoit la question en langage naturel, génère un SELECT borné, l'exécute et renvoie les supports correspondants.

Le PDF devient une table, la table devient interrogeable, et l'agent ne fait que traduire l'intention en SQL. La donnée reste vérifiable à chaque étape.

La méthode, étape par étape

Extraire les supportsDu PDF vers CSV, JSON, Excel et un schéma SQLite typé.
Enrichir par LLMClasse d'actifs, zones, thèmes, tickers, ISR, éligibilité, déduits de l'intitulé et de l'objectif de chaque fonds.
Construire la table interrogeableUne table à colonnes propres plus une vue enrichie, prête pour des requêtes.
Traduire la question en SQLL'agent transforme une demande en français en un SELECT borné, puis l'exécute.
Rendre les résultatsUn tableau statique pour parcourir, un point de terminaison de chat pour la recherche agentique.

Ce que ça montre

Un document produit n'est pas une base de données, mais on peut l'y ramener. Une fois la liste extraite et enrichie, la question de l'utilisateur pilote une requête bornée, ce qui reste bien plus fiable et vérifiable qu'une recherche par similarité sur du texte brut.

Livrables

La table des supports en CSV, JSON, Excel et SQLite, un tableau HTML pour parcourir, et une interface de chat pour la recherche par agent.

Démonstration statique : la table des 484 supports extraite du contrat, à parcourir et filtrer ; elle s'ouvre en pleine page.
